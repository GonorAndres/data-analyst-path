# Phase 6: Follow-Up and Iteration

## Goal

Close the loop. Measure whether recommendations were implemented, track their impact, and identify follow-up questions. The best analysts don't deliver and disappear.

## Follow-Up Checklist

After delivering an analysis:

- [ ] **Schedule a follow-up**: 2-4 weeks after delivery, check if recommendations were acted on
- [ ] **Track impact metrics**: If you recommended targeting cohort X, did the team do it? What happened?
- [ ] **Document learnings**: What worked in this analysis? What would you do differently?
- [ ] **Identify next questions**: Every analysis surfaces new questions. Log them for prioritization.
- [ ] **Update dashboards**: If the analysis led to a new KPI or view, add it to existing dashboards

## Impact Documentation (For Portfolio)

In your portfolio, you won't have real follow-up. Instead, simulate it:

### "Expected Impact" Section
```markdown
## Expected Impact

If the recommended retention campaign targets the 30-60 day cohort:
- Estimated churn reduction: 10-15% in that segment
- Revenue retained: $180K-$270K annually (based on average LTV of $120)
- Break-even point: Campaign cost < $90K/year

## Metrics to Monitor
- 30-day retention rate (baseline: 42%)
- Cohort-specific reactivation rate
- Cost per reactivated customer vs. acquisition cost
```

This shows hiring managers you think beyond the analysis itself.

## Lessons Learned Template

After each project, write a brief retrospective:

| Category | Notes |
|----------|-------|
| **What went well** | [e.g., "SQL-first approach was efficient for cohort calculation"] |
| **What was harder than expected** | [e.g., "Date timezone mismatches took 2 hours to debug"] |
| **What I'd do differently** | [e.g., "Profile data more thoroughly before starting analysis"] |
| **New skill developed** | [e.g., "First time using Power BI bookmarks for guided navigation"] |
| **Reusable pattern** | [e.g., "The cohort SQL pattern works for any time-based segmentation"] |

## Key Metrics for Portfolio Success (from Research)

Across all 6 projects, ensure:
- Every project has a clear business question and actionable recommendations
- At least 3 different tools are demonstrated (SQL, Python, R, Power BI, Streamlit)
- At least 1 project uses Mexican/LATAM data (leverages your background)
- At least 1 project leverages actuarial/insurance domain knowledge
- All interactive outputs are accessible via public URLs
- All code is reproducible with clear instructions
